## API Documentation — OpenAPI / Swagger

| Tool | Version | Purpose |
|---|---|---|
| OpenAPI Specification | 3.1 | API contract standard |
| Swagger UI | Latest | Interactive API explorer (dev/staging only) |
| Framework library | (see below) | Code-first doc generation |

**Framework setup:**
- **ASP.NET Core (.NET 9+):** Built-in `Microsoft.AspNetCore.OpenApi` + Swagger UI via `Swashbuckle.AspNetCore.SwaggerUI`
- **ASP.NET Core (.NET 8):** `Swashbuckle.AspNetCore` — `AddSwaggerGen()` + `UseSwaggerUI()`
- **FastAPI:** Built-in, configure `docs_url=None` in production
- **NestJS:** `@nestjs/swagger` — `SwaggerModule.setup()`
- **Express.js:** `swagger-jsdoc` + `swagger-ui-express`
- **Spring Boot:** `springdoc-openapi-starter-webmvc-ui` 2.x

**Mandatory rules:**
- Every public endpoint MUST have an OpenAPI annotation with `operationId`, parameters, and all response codes
- Swagger UI and `/openapi.json` endpoint MUST be disabled in production (environment guard)
- All APIs MUST implement `/health` and `/health/ready` endpoints
- Error responses MUST use a consistent `ErrorResponse` schema — never raw strings
- `operationId` follows `verbNoun` convention: `getUser`, `listOrders`, `createPayment`
- Datetime fields use ISO 8601 (`date-time` format); IDs document integer or UUID type

Full rules and code examples in: `.github/instructions/swagger-api-docs.instructions.md`
